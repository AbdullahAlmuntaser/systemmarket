## 2) تحليل الأنظمة الأساسية

### 🟢 المبيعات (Sales)
- **إنشاء فاتورة:** يعمل.
- **تعدد المنتجات:** متاح.
- **الحسابات:** يتم حساب السعر والضريبة والخصم.
- **المخزون/القيد:** مربوط بـ `TransactionEngine` و `PostingEngine`.

### 🟢 المشتريات
- **تعدد المنتجات:** متاح.
- **الوحدات:** يدعم الوحدات (كرتون، قطعة، إلخ) عبر `UnitConversionService`.
- **المخزون/القيود:** يتم إضافة للمخزون، تحديث `BuyPrice` (متوسط التكلفة)، وإنشاء قيود محاسبية عبر `PostingEngine`.

### 🟢 المخزون
- **Batches:** مدعوم عبر `ProductBatches` و `TransactionEngine`.
- **FEFO:** منطق الـ `TransactionEngine` يستخدم `batchId` للخصم.
- **Transactions:** يتم تسجيل كل حركة في `InventoryTransactions`.

### 🟢 المحاسبة
- **شجرة الحسابات:** موجودة في `GLAccounts`.
- **القيود:** يتم إنشاؤها عبر `PostingEngine` الذي يتحقق من التوازن (Debit = Credit).

## 3) تحليل الربط (Integration)

| العلاقة | الحالة | الملف المسؤول |
| :--- | :--- | :--- |
| المبيعات → المخزون | ✅ فعالة | `TransactionEngine` |
| المبيعات → المحاسبة | ✅ فعالة | `PostingEngine` |
| المشتريات → المخزون | ✅ فعالة | `PurchaseService` |
| المشتريات → المحاسبة | ✅ فعالة | `PurchaseService` -> `PostingEngine` |

---

## 4) تحليل قاعدة البيانات
- **الجداول:** النظام ضخم (أكثر من 50 جدولاً) تغطي ERP كامل.
- **العلاقات:** علاقات Foreign Keys قوية بين الفواتير، الأصناف، القيود، والمخازن.
- **مشاكل التصميم:** وجود جداول "مهجورة" أحياناً أو جداول قديمة (`posting_examples.dart` تم حذفه).

---

## 5) تحليل الخدمات (Services)
- **PostingEngine:** المحرك الأساسي للقيود (مكتمل).
- **AccountingService:** إدارة الحسابات والقيود العامة.
- **InventoryService:** إدارة العمليات المخزنية.
- **PurchaseService:** معالجة المشتريات (بعد الربط الأخير).

---

## 6) العمليات الكاملة (End-to-End)
- 🧪 **بيع:** (فاتورة → `TransactionEngine` → خصم `Batch` → تسجيل `SaleItem` → `PostingEngine` → قيد).
- 🧪 **شراء:** (فاتورة → `UnitConversion` → `MovingAverageUpdate` → `InventoryTransaction` → `PostingEngine` → قيد).

---

## 7) كشف النواقص
- **PriceHistory:** غير موجود (جدول `PriceHistory` غير مفعّل).
- **Payroll → Accounting:** لم يتم ربط الرواتب بالمحاسبة برمجياً حتى الآن.
- **تنبيهات انخفاض المخزون:** موجودة كمنطق، لكن تحتاج واجهة (UI) مخصصة للإشعارات الفورية.
